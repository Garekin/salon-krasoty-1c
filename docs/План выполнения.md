# План выполнения ТЗ: Салон красоты, часть 1 (MVP)

## Фаза 0. Оценка сложности

**Сложность: высокая.** Признаки:
- Внедрение БСП (3 обязательные + 3 дополнительные подсистемы) с правками типового кода.
- 11 справочников, 7 перечислений, 5 документов, 5 регистров накопления, 1 регистр сведений, 4 отчёта (СКД), 1 печатная форма.
- Неочевидные зависимости: документы → регистры → отчёты; ввод на основании; расчёт себестоимости «по средней»; контроль остатков; функциональная опция складов.

**Стратегия:** последовательная сборка по слоям метаданных (снизу вверх по зависимостям), с тестированием в 1С:Предприятие после каждого крупного блока. Базовые объекты создаём раньше тех, что на них ссылаются.

**Среда разработки:** Конфигуратор вручную. Роль ассистента — методическая поддержка: пошаговые инструкции по созданию объектов, точные настройки реквизитов/форм, готовый код модулей (проведение, обработчики форм, печатная форма), тексты запросов и настройки СКД, сверка результата с ТЗ. Объекты пользователь создаёт в Конфигураторе самостоятельно.

---

## Порядок зависимостей (почему такой)

```
ИБ + БСП → Интерфейс/стиль → Подсистемы → Перечисления → Группы команд
  → Справочники (Должности → Сотрудники → остальные)
  → Функц. опция Склады
  → Документы (Поступление, Предв.запись, Реализация, ПКО, РКО)
  → Регистры (сведения Цены; накопления)
  → Движения документов + проведение (контроль остатков, себестоимость)
  → Ввод на основании
  → Печатная форма Реализации
  → Отчёты (СКД)
  → Командный интерфейс + начальная страница
  → Заполнение данными + тестирование
```

## Сквозные требования (проверять на каждом этапе)

- Использовать механизмы БСП (`СообщитьПользователю`, `ЗначениеРеквизитаОбъекта` и др.).
- Не создавать объекты/функциональность сверх ТЗ.
- Суммовые реквизиты — число или определяемый тип; итоговые суммы недоступны для ручного ввода и пересчитываются.
- Формы должны соответствовать прототипам/макетам из ТЗ.
- Вся функциональность тестируется в режиме 1С:Предприятие.

## Методические якоря (уроки курса)

Диплом «Салон красоты» методически совпадает с учебной задачей «Барбершоп». Каждый блок опирается на конкретные уроки — реализовывать строго по ним.

| Блок | Уроки курса | Ключевой подход из урока |
|------|-------------|--------------------------|
| A (БСП) | **Спринт 10, Тема 4–5** | Объединение с файлом `1Cv8.cf` (НЕ `_demo`/`_international`); «Поставить на поддержку?» → **Да**; снять «Сохранять настройки автоматически»; обработки `ПервоеВнедрениеБСП.epf` и `ПроверкаВнедренияБСП`; обязательные подсистемы = база для всех; рефакторинг после внедрения |
| B (стиль/картинки) | **Спринт 5, Тема 4** | Картинки — в раздел *Общие картинки* (имена = подсистемы), привязка через свойство *Картинка* подсистемы; объект *Стили* → новый стиль с цветами ТЗ → основной стиль конфигурации |
| F4/F5 (ПКО/РКО) | **Спринт 4, Урок 10–11** | `УстановитьВидимость()` по виду операции; `ОграничениеТипа` для составного Плательщика/Получателя; `ПоказатьВопрос` при смене вида операции с очисткой реквизитов; вынос общей логики видимости договора в общий модуль |
| H (проведение) | **Спринт 4, Урок 8, 10; Спринт 11, Тема 2 Урок 4** | Старт — `ОбработкаПроведения()` в модуле объекта; **новая методика контроля остатков**: записать движения (`Движения.Записать()`) → запрос к виртуальной таблице `.Остатки()` на `Граница(МоментВремени())` с условием `Остаток < 0` → при непустом результате `Отказ = Истина` + `СообщениеПользователю` с номенклатурой и нехваткой |

| J (печать) | **Спринт 10, Тема 3 Урок 3–5; Тема 5 Урок 5** | Печать через подсистему Печать; данные организации — `ОрганизацииСервер.СведенияОбОрганизации(Организация)` → структура с Наименование/ИНН/Руководитель/ДолжностьРуководителя/ГлавныйБухгалтер; строки — `СтроковыеФункцииКлиентСервер.ПодставитьПараметрыВСтроку`; пустые ТЧ не выводить (`Если ВыборкаТЧ.Количество() Тогда`) |
| K (отчёты СКД) | **Спринт 3, Тема 4/5** | Стандартный период + быстрые отборы (по умолчанию выключены); макет оформления *Античный* (Настройки → Другие настройки → Макет оформления); группировки СКД вместо ИТОГИ; группировка по реквизиту через точку (Тип номенклатуры); отрицательные — красным через условное оформление |
| E (функц. опция) | **Спринт 4, Урок 11; Спринт 10, Тема 6 Урок 1** | Опция хранится в константе; чтение `Константы.ИспользоватьСклады.Получить()` один раз до цикла; опция (в отличие от константы Барбершопа) управляет видимостью объектов в интерфейсе — привязать к ней складские объекты |

> Замечание по составному типу: в уроках видимость и `ОграничениеТипа` реквизита (Плательщик/Получатель, Договор) меняются в обработчиках формы по виду операции и типу контрагента — переиспользовать этот паттерн в F4/F5.

## Приложение Б. Примеры кода по пунктам плана

Каркасы кода для пунктов, где требуется BSL. Примеры из учебных уроков курса (через `search_sprints`/`find_bsl_examples`) — адаптировать имена объектов под салон красоты. Источник указан для каждого.

### Б1. Контроль остатков «новой методикой» — блоки H1, H2 (проведение)
*Источник: Спринт 4, Урок 8/10.* Паттерн: записать движения → запросить остатки `< 0` на `Граница(МоментВремени())` → при непустом результате `Отказ = Истина` + сообщение.

```bsl
Процедура ОбработкаПроведения(Отказ, РежимПроведения)
    // ... формирование движений по ТоварыНаСкладах (Расход) ...
    Движения.ТоварыНаСкладах.БлокироватьДляИзменения = Истина;
    Движения.Записать();

    Запрос = Новый Запрос;
    Запрос.УстановитьПараметр("МоментВремени", Новый Граница(МоментВремени()));
    Запрос.Текст =
    "ВЫБРАТЬ
    |   ТоварыНаСкладахОстатки.Номенклатура.Представление КАК Номенклатура,
    |   -ТоварыНаСкладахОстатки.КоличествоОстаток КАК Нехватка
    |ИЗ
    |   РегистрНакопления.ТоварыНаСкладах.Остатки(&МоментВремени, ) КАК ТоварыНаСкладахОстатки
    |ГДЕ
    |   ТоварыНаСкладахОстатки.КоличествоОстаток < 0";

    Результат = Запрос.Выполнить();
    Если НЕ Результат.Пустой() Тогда
        Отказ = Истина;
        Выборка = Результат.Выбрать();
        Пока Выборка.Следующий() Цикл
            ОбщегоНазначения.СообщитьПользователю(
                СтрШаблон("Не хватает ""%1"" в количестве %2", Выборка.Номенклатура, Выборка.Нехватка));
        КонецЦикла;
    КонецЕсли;
КонецПроцедуры
```
> При функц. опции «Запрет отрицательных остатков»: читать `Константы.ИспользоватьСклады.Получить()` (или соответствующую опцию) ОДИН раз до цикла; при выключенной — не ставить `Отказ`, но сообщать об отрицательных остатках.

### Б2. Себестоимость «по средней» — блок H2 (Реализация)
*Источник: Спринт 4, Урок 8.* Запрос остатков ТоварыНаСкладах с ресурсами Количество и Сумма; себестоимость строки = `СуммаОстаток / КоличествоОстаток × СписываемоеКоличество`. В секции ИТОГИ по документу — агрегат `МАКСИМУМ(КоличествоВДокументе)` (не СУММА — иначе дублирование по числу партий), `СУММА(КоличествоОстаток)`. Остаточный регистр должен «закрываться в ноль»: если списываем партию целиком — берём её сумму без пересчёта (избегаем «зависших копеек»).

### Б3. Цена из периодического РС на дату — блоки F2, F3 (ТЧ при изменении номенклатуры)
*Источник: Спринт 4, Урок 6; листинги 5.8/5.9.* Внеконтекстная серверная функция + метод `ПолучитьПоследнее(Период, Отбор)`:

```bsl
&НаКлиенте
Процедура УслугиУслугаПриИзменении(Элемент)
    СтрокаТЧ = Элементы.Услуги.ТекущиеДанные;
    СтрокаТЧ.Цена = ПолучитьЦену(СтрокаТЧ.Услуга, Объект.Дата);
    ПересчитатьСтроку(СтрокаТЧ);
КонецПроцедуры

&НаСервереБезКонтекста
Функция ПолучитьЦену(Номенклатура, Дата)
    Отбор = Новый Структура("Номенклатура, ВидЦены", Номенклатура,
        Перечисления.ВидыЦенПродажи.Розничная);
    Структура = РегистрыСведений.ЦеныНоменклатуры.ПолучитьПоследнее(Дата, Отбор);
    Возврат Структура.Цена;
КонецФункция
```

### Б4. Отчёт «Финансовый результат» — блок K3
*Источник: Спринт 4, Урок 9.* Запрос из трёх ВТ: статьи затрат из `Справочник.Номенклатура`, обороты `РегистрНакопления.Продажи.Обороты` (Доходы), обороты `РегистрНакопления.УчетЗатрат.Обороты` (Расходы); левые соединения по статье затрат, `ЕСТЬNULL(..,0)`. Поле «Финансовый результат» = Доходы − Расходы; в СКД — условное оформление: >0 зелёным с префиксом «Прибыль:», <0 красным с «Убыток:». Полный текст запроса — в [tool-results](../../.claude/projects/Q------------------/b1b732e5-43af-49ea-928e-33c3b507fe58/tool-results/mcp-1c-docs-rag-find_bsl_examples-1780521228085.txt) (Спринт 4 Урок 9).

### Б5. Автор документа — блок F (все документы)

Процедура ПередЗаписью(Отказ, РежимЗаписи, РежимПроведения)

### Б8. Контроль ИНН — блок D8 (Организации)
Проверка длины ИНН по типу организации в `ОбработкаПроверкиЗаполнения` модуля объекта: ЮЛ → 10 символов, ИП → 12; при несоответствии — `ОбщегоНазначения.СообщитьПользователю(...)` + добавить в `ПроверяемыеРеквизиты`/`Отказ`.

### Б9. Формирование наименования — Сотрудники (ФИО), Договоры (№ от даты)
В `ОбработкаЗаполнения`/`ПередЗаписью`: `Наименование = СокрЛП(Фамилия) + " " + СокрЛП(Имя) + " " + СокрЛП(Отчество)`; для договора `Наименование = СтрШаблон("№%1 от %2 г", Номер, Формат(ДатаНачала, "ДЛФ=D"))`.
