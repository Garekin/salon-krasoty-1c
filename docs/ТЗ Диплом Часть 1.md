  

# Диплом: Салон красоты, часть 1

## Как выполнять работу

❗ **Соблюдайте правила и подходы, изложенные в уроках**

Дипломный проект основан на теории, изложенной в уроках.

Ваша задача — применить именно те принципы и подходы, которые были разобраны в теоретической части курса.

Строго следуйте стандартам, архитектурным решениям и методам, о которых рассказано в уроках — и тогда ваша работа порадует ревьюеров. А ревьюеры порадуют вас.

- Не создавайте объекты и функциональность (формы, элементы, команды), которых нет в техническом задании.
- Вся функциональность, которую вы разработаете, должна быть протестирована в режиме 1С:Предприятие.
- Если для реквизитов не указаны тип данных или иные настройки — установите их самостоятельно, исходя из смысла задачи.
- Заполните все справочники, создайте и проведите все документы. Создайте как минимум по три элемента для каждого объекта метаданных (например, для документа или справочника).
- Для каждого разрабатываемого объекта заведите пользовательские данные, соответствующие тематике работы.
- Если возникают вопросы или сомнения по выполнению работы — задавайте вопросы в чатах Пачки.

Вам предстоит самостоятельно разработать конфигурацию на платформе 1С. Разработка будет разделена на два этапа:

1. Разработать MVP-версию продукта: 
   - Создать механизмы оперативного учёта
   - Разработать несколько аналитических отчётов, используя библиотеку стандартных подсистем (БСП).
2. Расширить конфигурацию: 
   - Разработать бухгалтерскую подсистему
   - Создать внешнюю печатную форму
   - Разработать интеграционные механизмы на примере мобильной конфигурации для записи клиентов на услуги
   - Автоматизировать загрузку прайс-листа из файла Excel

**MVP**

Minimum Viable Product (минимально жизнеспособный продукт) — это самая ранняя версия продукта, у которой есть минимальный набор функций, достаточный для презентации публике и проверке на первых потребителях.

Для каждого из двух этапов подготовлено отдельное техническое задание. Второе техзадание станет доступно после того, как вы реализуете первую часть и успешно пройдёте ревью.

6. Разработка отчётов

**6.1. Отчёт по товарам на складах**

Разработайте отчёт **Товары на складах**, он должен показывать данные по номенклатуре (количество) в разрезе складов за указанный период. В отчёте должны быть данные 

- по остаткам на начало периода,
- по приходу за период,
- по расходу за период,
- по остатку на конец периода.

**Требования к отчёту:**

- Должен содержать Стандартный период, быстрые отборы по Организации, Складу и Номенклатуре.
- Для отчёта должен быть установлен макет оформления «Античный».
- Источник формирования отчёта — регистр Товары на складах.
- Отчёт необходимо вывести в интерфейс подсистем Покупки и Продажи.
- Отрицательные значения должны выделяться красным цветом.

Отчёт должен содержать группировки:

- Склад
- Тип номенклатуры (товар, материал)
- Номенклатура (детальные записи)

**Макет отчёта по товарам на складах:**

**6.2. Отчёт по продажам**

В конфигурации необходимо разработать отчёт **Продажи***,* который будет показывать информацию о продажах услуг салона.

**Требования к отчёту:**

- Должен содержать Стандартный период, быстрые отборы по Организации, Мастеру (сотрудник), Клиенту (контрагент), Должности (должность сотрудника), Услуге (номенклатура).
- Для отчёта должен быть установлен макет оформления «Античный».
- Источник формирования отчётов — регистр Продажи.
- В отчёте должны выводиться данные только по номенклатуре с типом Услуга.
- Отчёт необходимо вывести в интерфейс подсистемы Продажи.

Отчёт должен содержать группировки:

- Мастер (Сотрудник).
- Клиенты (Контрагенты).
- Услуга (Номенклатура).
- Регистратор.

Отчёт должен содержать промежуточные итоги по группировкам и общий итог. Сортировка — по убыванию суммы.

**6.3. Отчёт по финансовому результату**

Для расчёта выручки и прибыли реализуйте отчёт, который будет собирать данные из регистров Продажи и Учёт затрат.

В отчёте должно быть три раздела:

- **Доходы** — сведения о выручке в разрезе статей затрат (из справочника Номенклатура) с общим итогом по разделу.
- **Расходы** — данные по затратам из регистра Учёт затрат в разрезе статей затрат с общим итогом по разделу.
- **Финансовый результат** (доходы минус расходы за указанный период). При положительном финансовом результате текст должен быть зелёным, а перед суммой должно выводиться слово «Прибыль:». При отрицательном финансовом результате текст должен быть красным, а перед суммой нужно вывести слово «Убыток:».
- Отчёт необходимо вывести в интерфейс подсистемы Продажи.

Данные должны формироваться за указанный период (стандартный период). Должен быть реализован отбор по Организации и Статье затрат.

**6.4. Отчёт по движению денег в кассах**

- Выбор периода отчёта необходимо реализовать с использованием Стандартного периода.
- В быстрых отборах должен быть отбор по Организации и Кассе.
- Макет оформления отчёта — «Античный».
- В отчёте должны быть выведены общие итоги.
